This method is done to help you to find the right path toward the template file, the repository and the temporary directory.
public string Template::display void [+]
This method displays the final rendered.
That allows you to check if the render has been done correctly or to displays the calculated content.

public string Template::render void [+]
This method trigger the file calculation between the sent data and the template.
Be aware, the "render" method only serve to generate the file.
To see the document strictly speaking, use the method "display"
public string Template::sendMail void [+]
This method sends by email the content of the calculated document.
So that mean you must use this method after the "render" method.
Moreover, you must define the email parameters before
She is useful when you generate reports which you want to receive on your email software.

Declaration of the repository folder for the rendered files.

public boolean Template::set_output_name string $name [+]
Declaration of the file name to the output file.
The name must contain the file extension.
Example, you want a HTML file, the name will be "my_file_name.html".
public boolean Template::set_render_type string $type [+]
This method allows you to define the type of the render. If you want to generate document, the appropriate render type is "permanent".
If you use template only to display something, the appropriate render type is "temporary".
public boolean Template::set_template_name string $source [+]
Declaration of the path where the template is.
The name contain the folder path and / or the template file name (includes the file extension)
public boolean Template::set_temporary_folders_path string $tmp_folders_path [+]
Declaration of the path of the temporary repository.
The folder path name do not must contain the final slash.
This is an optional parameter.
public boolean Template::set_utf8_read_traitment string $traitement [+]
Declaration of the UTF8 treatment for the reading process of the output file.
The treatment possibilities are "none", "encode" and "decode".
"none" is the default setting.
"none" mean "use the default encoding of the output file"
public boolean Template::set_utf8_write_traitment string $traitement [+]
Declaration of the UTF8 treatment for the writing processing of the output file.
The treatment possibilities are "none", "encode" and "decode".
"none" is the default setting.
"none" mean "use the default encoding of the template file"
public boolean Template::set_vars array $vars [+]
Declaration of simples variables for the render
The input variable must be an associative arrays where keys represents the variable name.
public boolean Template::set_vars_delim string $delim [+]
Declaration of the delimiter of variables.
The delimiter is the character which surround the variable name.
This allows the programme to identify the place where you need the expected value.
public string Template::show_warnings void [+]
This method displays all encountered errors during the rendering.
These errors are not blocking errors.
